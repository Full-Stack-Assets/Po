Constraint-Optimized Orchestration

The COO Engine

A constraint-aware orchestrator for multi-agent LLM systems — routing every task to the right agent under hard budget, latency, and capacity limits.

▶ Launch simulation ◉ Connect live backend View on GitHub

The simulation runs entirely in your browser — no API key or backend required. The live console wires to a running orchestrator (/v2/status, /v2/health, /v2/approvals) and drives the human-in-the-loop approval queue.

Core capabilities

◈

Multi-dimensional fitness scoring

Every (agent, task) pair is scored across five weighted constraints: type match, cost, latency, load, and budget headroom.

⊕

Intelligent fallback routing

Evaluates cross-type routing and uses it when it is genuinely the best option — not limited to same-type matching.

⟨⟩

Predictive spawn

Forecasts cost before scaling up, only spawning agents when demand and the budget both justify it.

≡

Cost-aware retirement

Retires idle agents under budget pressure or after an idle timeout, keeping spend inside hard limits.

◉

Full audit trail

Every routing, spawn, and retirement decision is logged with a timestamp, score, and human-readable justification.

⌗

Mini CLI & analytics

Inspect agent stats, queue depth, budget, health, and decisions from the command line with mini.
